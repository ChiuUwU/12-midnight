# 十二点天黑 MVP 开发清单 v0.1

## 1. MVP 原则

第一版先以手机网页版做可用闭环，不追求完整自动判定。规则复杂处允许法官确认，但系统必须记录结果。微信小程序版本暂时作为后续选择。

优先级从高到低：

1. 房间创建和加入。
2. 座位绑定。
3. 固定版型发牌。
4. 玩家私密看身份。
5. 法官看全身份。
6. 夜间流程记录。
7. 白天投票和 PK。
8. 死亡与复盘记录。

## 2. 数据模块

### 2.1 房间

字段：

- 房间号。
- 模式：有法官 / 无法官。
- 当前版型。
- 当前阶段。
- 当前轮次。
- 法官用户。
- 玩家列表。
- 操作日志。

### 2.2 玩家

字段：

- 用户标识。
- 昵称。
- 座位号。
- 真实身份。
- 当前阵营。
- 是否存活。
- 是否警长。
- 是否已查看身份。
- 技能状态。
- 临时标记。

### 2.3 版型

字段：

- 版型 ID。
- 名称。
- 玩家数。
- 身份池。
- 夜间流程。
- 特殊规则配置。

### 2.4 操作日志

记录：

- 发牌。
- 查看身份。
- 夜间行动。
- 死亡结算。
- 投票。
- PK。
- 放逐。
- 遗言。
- 警徽处理。
- 死亡技能。
- 复盘开启。

## 4. 第一阶段功能任务

### 4.1 房间

- 创建房间并生成 6 位房间号。
- 支持有法官和无法官模式。
- 支持加入房间。
- 支持离开房间。
- 支持恢复当前房间。

验收：

- 法官创建房间后能看到房间号。
- 玩家输入房间号能进入同一房间。
- 房间内能看到 1-12 号座位状态。

### 4.2 座位

- 玩家选择座位。
- 已占用座位不可选。
- 发牌前允许换座。
- 发牌后锁定座位。

验收：

- 12 个座位不会重复绑定。
- 发牌后无法换座。

### 4.3 版型

- 内置四个固定版型。
- 展示版型身份组成和关键规则。
- 法官发牌前可切换版型。

验收：

- 四个版型均可选择。
- 每个版型生成正确数量的玩家身份。
- 盗宝大师版型能正确生成盗宝牌堆。

### 4.4 发牌

- 满 12 人后允许发牌。
- 系统随机分配身份。
- 玩家只能看到自己的身份。
- 法官能看到全部身份。

验收：

- 发牌后 12 名玩家身份不重复异常。
- 玩家端无法访问其他玩家身份。
- 法官总览显示完整身份表。

### 4.5 身份查看

- 玩家身份页显示身份名、阵营和技能摘要。
- 玩家确认查看后记录状态。
- 盗宝大师能看到盗宝牌堆。
- 机械狼模仿后能看到模仿结果。

验收：

- 普通玩家只能看到一张身份牌。
- 特殊身份有必要的额外信息。

### 4.6 夜间流程

- 根据版型生成夜间步骤。
- 法官或公共手机选择行动目标。
- 支持狼人空刀。
- 预言家、通灵师不允许空验。
- 守卫不可连续守同一人。
- 摄梦人必须每晚摄一人。
- 女巫记录解药和毒药使用状态。

验收：

- 每步只能选择合法目标。
- 不合法操作有明确提示。
- 操作完成后写入日志。

### 4.7 白天投票

- 记录每名存活玩家投票目标。
- 计算最高票。
- 平票进入 PK。
- PK 再平票则无人出局。
- PK 发言顺序按白天发言顺序反向生成。

验收：

- 单一最高票能进入放逐。
- 平票能进入 PK。
- PK 再平票不会放逐任何人。

### 4.8 死亡与复盘

- 记录死亡玩家、死亡阶段、死亡原因。
- 支持夜死无遗言、白天放逐有遗言。
- 支持警徽移交或撕毁。
- 支持手动确认死亡技能是否发动。
- 支持游戏结束后复盘。

验收：

- 复盘能看到身份、阵营、死亡、投票、夜间行动。
- 法官手动确认事项有日志。

## 5. 建议技术实现顺序

1. 先做本地单机状态版本。
2. 完成规则配置和发牌逻辑。
3. 完成房间、座位、身份页。
4. 完成法官总览和复盘。
5. 再接入云端房间同步。
6. 最后做夜间流程与投票自动化。

理由：

- 发牌和身份查看是最小可用核心。
- 规则配置越早落地，后续页面越不容易返工。
- 云同步会增加调试成本，不适合放在第一步。

## 6. 第一版不做

- 不做自由自定义版型。
- 不做玩家个人手机夜间私密操作。
- 不做完整自动胜负判断。
- 不做在线语音房。
- 不做复杂反作弊检测。
- 不做战绩系统。

## 7. 下一步开发入口

当前已转为网页版优先。建议先实现：

- `boards`：四个版型配置。
- `roles`：身份元信息。
- `deal`：发牌函数。
- `room`：房间和座位状态。
- `identity`：玩家身份展示。

完成这些后，就可以先跑通“创建房间 -> 加入座位 -> 选择版型 -> 发牌 -> 看身份 -> 法官总览”。
