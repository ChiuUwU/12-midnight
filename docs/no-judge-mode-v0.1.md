# 无法官模式 v0.1

## 定位

无法官模式仍然服务线下狼人杀，不把发言、上警、投票和唱票搬到线上。

- 玩家手机：私密看牌、仅在轮到自己身份时提交夜间行动。
- 公共控制设备：发牌、开始夜晚、语音播报、公布公开死讯、记录白天最终出局结果。
- 服务端：校验行动权限、推进步骤、统一结算、保存刷新恢复所需状态。

## 权限边界

- 无法官房主拥有 `isController`，但 `isJudge` 永远为 `false`。
- 控制设备不能占 1-12 号座位，也不能获取身份总览、夜间行动或死亡原因。
- 普通玩家只获取自己的身份。
- 等待玩家只能看到通用等待状态，不获取当前行动角色和台词。
- 当前身份玩家只获取当前一步和必要的私密上下文。
- 狼队步骤允许当前版型中会进入狼队的存活角色共同操作，服务端只接受第一个有效统一刀口。

## 已实现流程

1. 控制设备创建 `SYSTEM` 房间并分享链接。
2. 12 名玩家各自在手机选择座位。
3. 控制设备发牌，但看不到任何身份。
4. 玩家私密查看自己的身份。
5. 控制设备开始夜晚并按步骤自动语音播报。
6. 对应身份玩家在自己手机提交行动；其他玩家提交会被服务端拒绝。
7. 控制设备轮询步骤，身份提交后自动播报下一步。
8. 预言家、通灵师、机械查验、机械模仿、假面验舞池结果只随提交响应返回本人。
9. 女巫本人在有解药时只获取当夜刀口。
10. 所有步骤完成后，控制设备结算并公布天亮死讯。
11. 蒙面延迟死亡和死亡技能只向对应玩家本人显示待办。
12. 公开死亡、开枪和白天放逐通过公共事件由控制设备自动播报。
13. 控制设备只记录线下白天产生的最终出局结果或无人出局；未记录不能进入下一夜。
14. 游戏结束后公开完整身份和复盘记录。

## 刷新策略

- 控制设备和等待玩家每 2.5 秒刷新房间状态。
- 当前行动玩家暂停自动刷新，避免清空尚未提交的号码选择。
- 提交后恢复刷新。
- 已播报的公共事件 ID 保存在浏览器本地，刷新页面不会重复播报旧死讯。

## 当前限制

- 浏览器语音依赖 Web Speech API；若系统禁止自动播放，可点击“重播当前台词”。
- 暂不处理行动超时、控制设备迁移和多设备争抢控制权。
- 暂不自动判断胜负；控制设备可随时结束游戏并记录获胜阵营。
- 复杂版型已共享有法官模式的规则引擎，但后续仍需增加第二夜及末局专项集成测试。
