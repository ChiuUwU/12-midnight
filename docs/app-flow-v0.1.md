# 十二点天黑页面流程 v0.1

## 1. 第一版产品边界

第一版优先做“能稳定开局、发牌、看身份、辅助法官记录流程”。暂不追求完全自动法官结算。

核心目标：

- 法官能创建房间。
- 玩家能加入房间并绑定座位号。
- 系统能按固定版型随机发牌。
- 玩家只能看自己的身份。
- 法官能看全部身份。
- 法官或公共手机能记录夜间行动。
- 白天能记录投票、PK、放逐、遗言和警徽处理。
- 游戏结束后能复盘身份和关键操作。

## 2. 用户角色

### 2.1 法官

法官拥有最高权限：

- 创建房间。
- 选择版型。
- 开始发牌。
- 查看全部身份。
- 控制游戏阶段。
- 记录夜间行动。
- 记录投票和死亡。
- 开启复盘。

### 2.2 玩家

玩家权限受限：

- 加入房间。
- 绑定座位号。
- 查看自己的身份。
- 查看公开信息，例如当前阶段、死亡玩家、警长、复盘信息。
- 第一版不在个人手机上执行夜间操作。

### 2.3 系统法官

无法官模式下，系统承担流程引导：

- 按版型提示夜间行动顺序。
- 在公共手机上收集行动目标。
- 每个身份行动结束后隐藏信息。
- 不提供全身份查看入口。

## 3. 页面清单

### 3.1 首页

入口：

- 创建房间
- 加入房间
- 继续上次房间

关键状态：

- 如果本机已有未结束房间，显示“继续房间”。
- 如果没有历史房间，只显示创建和加入。

### 3.2 创建房间页

字段：

- 模式：有法官 / 无法官
- 版型选择
- 房间名称，可选

操作：

- 创建房间
- 返回首页

创建成功后：

- 生成房间号。
- 当前用户成为法官或房主。
- 进入房间等待页。

### 3.3 加入房间页

字段：

- 房间号
- 昵称，可选

操作：

- 加入房间
- 返回首页

加入成功后：

- 进入座位号选择页。

### 3.4 房间等待页

法官视图：

- 房间号。
- 当前版型。
- 已加入玩家列表。
- 1-12 号座位状态。
- 开始发牌按钮。
- 调整版型入口。
- 踢出玩家或清空座位入口。

玩家视图：

- 房间号。
- 当前座位号。
- 当前人数。
- 等待法官发牌。

限制：

- 未满 12 人时，默认不允许开始发牌。
- 如后续支持代填空座，可另加“补齐虚拟玩家”。

### 3.5 座位号选择页

内容：

- 1-12 号座位。
- 已占用座位不可选。
- 玩家选择一个座位并确认。

规则：

- 每名玩家只能绑定一个座位。
- 换座需要先释放原座位。
- 发牌后不可换座。

### 3.6 版型选择页

固定版型：

- 预女猎白混
- 假面舞会
- 盗宝大师
- 机械狼通灵师

每个版型展示：

- 人数。
- 阵营组成摘要。
- 关键特殊规则。
- 是否有复杂额外牌池。

第一版不提供自由自定义版型，只保留后续入口。

### 3.7 玩家身份页

内容：

- 座位号。
- 身份名称。
- 阵营或胜利条件。
- 简短技能说明。

安全规则：

- 玩家只能查看自己的身份。
- 不显示他人身份。
- 不显示完整发牌列表。
- 进入夜间操作时，玩家身份页不参与操作。

特殊身份显示：

- 混血儿：显示“首夜选择榜样后，胜利条件跟随榜样阵营”。
- 盗宝大师：显示盗宝牌堆和当前阵营。
- 机械狼：显示模仿目标和模仿身份，仅在模仿后显示。
- 假面：显示验证和面具限制。

### 3.8 法官总览页

内容：

- 12 个座位。
- 每个座位的玩家名、身份、阵营、生死状态。
- 当前游戏阶段。
- 当前夜晚或白天轮次。
- 夜间流程入口。
- 白天投票入口。
- 死亡结算入口。
- 复盘入口。

权限：

- 只有法官模式下的法官可以访问。
- 无法官模式下不提供该页面。

### 3.9 夜间流程页

目标：

- 按版型生成当前夜晚行动顺序。
- 法官或公共手机逐步记录行动。

通用组件：

- 当前步骤标题。
- 语音或文字提示。
- 可选目标座位。
- 确认按钮。
- 跳过按钮，仅允许可空过身份使用。
- 上一步修正入口，仅法官模式允许。
- 完成后遮罩。

隐私规则：

- 公共手机每步完成后进入遮罩。
- 下一身份行动前不展示上一身份目标。
- 无法官模式不展示全局结果，只记录到系统状态。

### 3.10 天亮结算页

内容：

- 昨夜死亡玩家。
- 死亡原因，仅法官可见；玩家端默认只显示公开死亡信息。
- 是否有特殊延迟死亡。
- 是否触发死亡技能。
- 是否进入白天发言。

玩家端：

- 显示公开死亡名单。
- 不显示被刀、被毒、被摄等细节。

### 3.11 白天投票页

内容：

- 存活玩家列表。
- 当前警长。
- 投票目标选择。
- 投票结果。
- 是否平票。

平票处理：

- 进入 PK。
- 自动列出平票玩家。
- 按“白天后发言者先发言”的规则生成 PK 发言顺序。
- PK 再投票。
- 再平票则无人出局。

### 3.12 放逐与遗言页

内容：

- 被放逐玩家。
- 是否有遗言。
- 是否能发动死亡技能。
- 白痴被放逐时公布身份并出局。
- 警长死亡时选择移交警徽或撕毁警徽。

技能限制：

- 猎人最后一神不能开枪。
- 狼王、机械猎人、盗宝猎人作为狼人阵营最后一狼不能开枪。
- 被毒杀不能开枪。

### 3.13 复盘页

内容：

- 全部座位身份。
- 最终阵营。
- 生死状态。
- 夜间行动记录。
- 投票记录。
- 关键死亡原因。
- 胜负结果。

开启条件：

- 法官手动开启。
- 或系统判断游戏结束后开启。

## 4. 主流程

### 4.1 有法官模式主流程

1. 法官创建房间。
2. 法官选择版型。
3. 玩家加入房间。
4. 玩家绑定座位号。
5. 法官确认 12 人满员。
6. 法官发牌。
7. 玩家查看自己的身份。
8. 法官进入总览页。
9. 法官开始第一夜。
10. 法官按夜间流程记录行动。
11. 系统生成天亮结算。
12. 白天发言。
13. 法官记录投票。
14. 放逐、遗言、警徽和死亡技能结算。
15. 判断胜负。
16. 未结束则进入下一夜。
17. 游戏结束进入复盘。

### 4.2 无法官模式主流程

1. 房主创建无法官房间。
2. 玩家加入房间。
3. 玩家绑定座位号。
4. 系统发牌。
5. 玩家查看自己的身份。
6. 公共手机进入夜间流程。
7. 系统逐步提示身份行动。
8. 当前身份玩家在公共手机上选择目标并确认。
9. 每步结束后公共手机遮罩。
10. 系统生成天亮公开死亡信息。
11. 玩家线下发言。
12. 公共手机记录投票。
13. 系统处理 PK、放逐和警徽。
14. 判断胜负。
15. 未结束则进入下一夜。
16. 游戏结束进入复盘。

## 5. 状态机

推荐房间阶段：

```ts
type RoomPhase =
  | "WAITING"
  | "DEALT"
  | "NIGHT"
  | "DAY_ANNOUNCEMENT"
  | "DAY_SPEECH"
  | "VOTE"
  | "PK"
  | "EXILE"
  | "LAST_WORDS"
  | "SHERIFF_TRANSFER"
  | "DEATH_SKILL"
  | "GAME_OVER"
  | "REVIEW";
```

阶段转换：

- `WAITING -> DEALT`：法官或系统发牌。
- `DEALT -> NIGHT`：开始第一夜。
- `NIGHT -> DAY_ANNOUNCEMENT`：夜间行动完成并结算。
- `DAY_ANNOUNCEMENT -> DAY_SPEECH`：公布死亡后进入发言。
- `DAY_SPEECH -> VOTE`：发言结束。
- `VOTE -> PK`：出现平票。
- `VOTE -> EXILE`：产生单一最高票。
- `PK -> EXILE`：PK 后产生单一最高票。
- `PK -> NIGHT`：PK 再平票，无人出局，进入下一夜。
- `EXILE -> LAST_WORDS`：被放逐玩家有遗言。
- `EXILE -> SHERIFF_TRANSFER`：警长死亡。
- `EXILE -> DEATH_SKILL`：死亡玩家可发动技能。
- `EXILE -> NIGHT`：无后续结算且游戏未结束。
- `任意结算阶段 -> GAME_OVER`：触发胜负。
- `GAME_OVER -> REVIEW`：开启复盘。

## 6. 第一版可以手动处理的事项

第一版可以先由法官手动确认，后续再自动化：

- 胜负判断。
- 复杂死亡技能是否可发动。
- 盗宝蒙面狼特殊结算。
- 舞池死亡结算。
- 机械狼技能刀触发。
- 混血儿榜样阵营变化。

这些事项即使手动处理，也必须记录在操作日志中，方便复盘。
