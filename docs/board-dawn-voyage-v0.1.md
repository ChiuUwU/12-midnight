## 3. 海妖 (siren)

### 3.1 基础属性
- 阵营：狼人
- 与其他狼人见面，参与狼队讨论和刀人决策
- 每晚额外独立选择风向

### 3.2 风向技能
每晚必须从以下三种风向中选择一种：

| 风向 | ID | 效果 |
|------|-----|------|
| 无风 | `calm` | 所有技能目标不发生偏移 |
| 顺风 | `tailwind` | 所有技能目标号码 +1（2号→实际3号） |
| 逆风 | `headwind` | 所有技能目标号码 -1（2号→实际1号） |

**限制**：不可连续两晚选择相同风向。

**受风向影响的技能**：狼刀、女巫毒药、预言家查验。`[待确认]` 女巫解药是否受风向影响？

**死亡后**：海妖死亡后，风向固定为死亡前最后一晚选择的风向，不再改变。

### 3.3 风向与座位循环
> `[待确认]` 12人圆桌，号码偏移是否循环？
> - 顺风 12号→1号？（12+1=13，13→1？）
> - 逆风 1号→12号？（1-1=0，0→12？）

## 4. 船长 (captain)

### 4.1 基础属性
- 阵营：好人神职
- 第二晚起每晚安必须选择一名存活玩家登船
- `[待确认]` 船长能否选择自己登船？
- `[待确认]` 船长是否可以空过（不选人）？

### 4.2 登船效果 — 有风（顺风/逆风）

船上玩家：
- 不受风向影响（技能目标为该玩家时不发生偏移）

溺亡：
- 登船玩家「上风向」相邻的一名玩家溺亡
- 顺风：登船 N 号 → N-1 号溺亡
- 逆风：登船 N 号 → N+1 号溺亡
- 溺亡为独立死亡原因，与狼刀、毒药并列

海妖和船长免疫溺亡。

> `[待确认]` 溺亡的结算优先级？溺亡是否可被女巫解药救？是否可被守卫守护免除？

### 4.3 登船效果 — 无风

船上玩家：
- 免疫狼刀
- 不免疫毒药
- 不会被奶穿（同守同救不会导致死亡）
- 无人溺亡

### 4.4 举例（顺风，船长登船2号）

| 原始目标 | 风向偏移 | 实际目标 | 结果 |
|----------|----------|----------|------|
| 刀1号 | 1→2 | 2号 | 2号在船上（不受风向），但因刀直接指向了2号的原始号码... `[待确认]` |
| 刀2号 | 无偏移 | 2号 | 2号在船上不受风向影响，2号被刀死亡 |
| 刀3号 | 3→4 | 4号 | 4号被刀死亡 |
| 验2号 | 无偏移 | 2号 | 查验2号底牌 |
| 毒2号 | 无偏移 | 2号 | 2号中毒 |

同时：1号溺亡。

> `[待确认]` 上面的"刀1号，实际为2号死"——我理解是：刀1号→顺风偏移→1→2→2号是实际受伤者→2号虽然在船上，但"不受风向影响"只意味着指向2号的技能不会被风吹走，不代表2号无敌？还是溺亡已经覆盖了1号，所以1号溺死+2号是刀的真正目标？

实际上让我重新整理你的例子：

你给的例子：
- 顺风，船长选2号上船 → 1号溺亡
- 刀1号 → 实际2号死
- 刀2号 → 2号不受风向影响，2号死
- 刀3号 → 因为顺风，实际4号死

我的理解：
- 刀1号：狼人原始选1号，顺风+1→2号，2号在船上但「不受风向」指的是以2号为目标的技能不被偏移，而刀1经过偏移后落在2号身上，所以2号死了。**也就是说"不受风向影响"只阻止偏移不阻止伤害。**
- 刀2号：狼人选2号，2号在船上不受风向→不偏移，2号被刀死。**对，不受风向=不被偏移，但该吃的伤害照样吃。**
- 刀3号：狼人选3号，顺风+1→4号，4号死。

**所以船上玩家的"不受风向影响"=指向TA的技能不会因为风而偏移到别人身上。但该技能如果经过（别人的）偏移后落在TA身上，或者直接指向TA，TA正常受伤。有风时船上玩家没有任何伤害减免，只是免疫偏移。**

同时溺亡独立发生：顺风登船2号→1号溺亡。

> `[待确认]` 以上理解是否正确？

## 5. 风向宣布

每天白天发言前，法官宣布"昨夜X风"。

| 船长状态 | 宣布方式 |
|----------|----------|
| 船长存活 | 船长宣布 |
| 船长死于当晚 | 法官代为宣布昨夜风向 |
| 船长死于前一白天 | 无人宣布，下一晚风向未知 |

> `[待确认]` "风向未知"意味着玩家不知道海妖选了什么风，但风向效果仍然生效（因为海妖还在选）？只是不公开而已？

## 6. 夜间流程顺序

```
夜晚 N：
1. 海妖选择风向（不能与昨晚相同；若海妖已死则风向固定不变）
2. 船长选择登船玩家（第二晚起，必须选一人）
3. 系统计算溺亡（有风时）
4. 狼人选择刀口（海妖参与讨论）
5. 女巫行动（看到狼人原始刀口号码）
6. 预言家查验
7. 系统进行风向偏移计算 + 结算死亡
```
